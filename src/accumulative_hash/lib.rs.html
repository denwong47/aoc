1//! # Accumulative Hash / Additive Commutative Hash
2//! 
3//! `accumulative_hash` provides an efficient, order-independent hashing mechanism ideal for
4//! tracking the state of additive data structures, such as paths in a Depth-First Search (DFS)
5//! where the order of nodes visited does not change the identity of the path's *set* of nodes.
6//! 
7//! This library implements **Commutative Hashing** using modular arithmetic (`wrapping_add`/`wrapping_sub`),
8//! guaranteeing that the order of insertion does not affect the final hash state.
9//! 
10//! ```text
11//! H({A, B, C}) = H({A, C, B}) = H({C, B, A})
12//! ```
13//! 
14//! This is achieved by pre-mixing each input value (`u32`, `u64`, etc.) into a high-quality, large
15//! pseudorandom number, and then combining these numbers via modular addition.
16//! 
17//! ## Features
18//! 
19//! * **Order-Independent Hashing (Commutative):** No need to sort input elements ([`Vec<u32>`]) before hashing, saving ``O(N \log N)`` per check.
20//! * **Incremental Updates:** ``O(1)`` addition (`add`) and removal (`remove`) of elements.``
21//! * **Composition Property:** Hashes are associative, meaning two accumulated hashes can be summed to get the hash of the combined set of elements.
22//! * **Thread Safety:** The [`AtomicAccumulativeHash`] struct uses [`std::sync::atomic::AtomicU64`], etc., with Compare-And-Swap (CAS) loops for lock-free, thread-safe updates to the hash state.
23//!   * **NOTE**: This requires the ``atomic`` feature flag to be enabled.
24//! * **Collision Resistance:** By default, uses large integer types (recommended [`u64`] and [`u128`]) and carefully chosen mixing constants derived from mathematical principles (like the Golden Ratio constant `0x9E3779B97F4A7C15F39CC0605CEDC834` for `u128`) to ensure a statistically low collision rate.
25//! 
26//! ## Example: DFS Path Tracking
27//! 
28//! This library solves the problem of tracking visited, order-independent paths in a DFS without expensive memory allocations or sorting.
29//! 
30//! ```rust
31//! use accumulative_hash::AccumulativeHash;
32//! use std::collections::HashSet; // other hash set implementations can be used as well
33//! 
34//! // Assuming your step IDs are u32
35//! type PathHash = u64;
36//! 
37//! fn check_and_mark_path(
38//!     visited: &mut HashSet<PathHash>,
39//!     current_path_hash: PathHash,
40//! ) -> bool {
41//!     // True if we've visited this state before, regardless of the order of steps.
42//!     !visited.insert(current_path_hash)
43//! }
44//! 
45//! fn depth_first_search(steps: &[u32]) {
46//!     let mut visited_states = HashSet::<PathHash>::default();
47//!     let mut current_hash_state = AccumulativeHash::<PathHash>::new();
48//! 
49//!     for &step_id in steps {
50//!         // 1. Add the step: O(1)
51//!         current_hash_state.add(step_id);
52//! 
53//!         let path_hash = *current_hash_state.state();
54//! 
55//!         // 2. Check for visit: O(1) average
56//!         if check_and_mark_path(&mut visited_states, path_hash) {
57//!             println!("Path leading to {:X} was already visited!", path_hash);
58//!             // Backtrack logic would follow here...
59//!         }
60//!         
61//!         // ... continue DFS ...
62//! 
63//!         // 3. Backtrack (Remove step): O(1)
64//!         // If we were using a recursive DFS, this would happen on return:
65//!         // current_hash_state.remove(step_id);
66//!     }
67//! }
68//! ```
69//! 
70//! ## Available Hash Types
71//! 
72//! The core trait for underlying types is [IsAccumulativeHashType], implemented for:
73//! 
74//! * [u8], [u16], [u32] (Not recommended for production due to high collision risk)
75//! * [u64] (Good balance of speed and collision resistance)
76//! * [u128] (Highest collision resistance)
77//! * [usize] (Based on the target platform's pointer width)
78//! 
79//! For thread-safe operations, use the [`AtomicAccumulativeHash`] struct with the ``atomic`` feature enabled.
80//! The underlying atomic types supported are:
81//! 
82//! * [AtomicU8] (Not recommended for production due to high collision risk)
83//! * [AtomicU16] (Not recommended for production due to high collision risk)
84//! * [AtomicU32] (Not recommended for production due to high collision risk)
85//! * [AtomicU64] (Recommended for thread-safe operations with good collision resistance)
86//! * [AtomicUsize] (Based on the target platform's pointer width)
87//! 
88//! [AtomicU8]: std::sync::atomic::AtomicU8
89//! [AtomicU16]: std::sync::atomic::AtomicU16
90//! [AtomicU32]: std::sync::atomic::AtomicU32
91//! [AtomicU64]: std::sync::atomic::AtomicU64
92//! [AtomicUsize]: std::sync::atomic::AtomicUsize
